Welcome to the Mock Privacy Center — a simulated environment built to assist developers, testers, and product teams in verifying Facebook integration flows, consent UI patterns, and policy display logic. 1. Information We Collect (Testing Context)

For demonstration and compatibility testing with Facebook Login, Graph API mocks, and Meta Business tools, this test environment does not store real user data. However, for realistic rendering, the following mock data categories may appear in logs or in-memory state during a testing session:

  • Mock profile data: simulated name, email hash, profile picture placeholder (test user id: 123456789_test).
  • Interaction signals: button clicks, policy acceptance timestamps (local session only).
  • Device & browser info: user agent, viewport size, preferred language – stored ephemerally.
  • Facebook test user tokens: synthetic OAuth tokens generated by Facebook's test user infrastructure (managed via Meta developer tools).

No sensitive personal information (government IDs, financial data, health information) is ever requested or processed in this mock policy.

2. How We Use Mock Information

Any mock data collected is used strictly for the following simulation purposes:

  • ✅ Validate that Facebook privacy interfaces render correctly.
  • ✅ Test consent management flows (e.g., "Accept", "Reject", "Learn More").
  • ✅ Simulate data deletion / download requests to align with Meta review requirements.
  • ✅ Debug UI components that reference legal documents inside Facebook Canvas apps or Page tabs.

After each test session (or within 24 hours), all simulated data is automatically discarded. No persistent storage is utilized.

3. Sharing & Third Parties (Mock Simulation)

Because this is a testing-only environment, we do NOT share mock data with external advertisers, analytics providers, or data brokers. However, to emulate realistic Facebook platform behavior, certain simulated interactions may involve:

  • Facebook Test Infrastructure – if you are using Facebook's official test users, limited app-scoped identifiers may be passed to Meta servers for OAuth validation. Refer to Meta’s actual Data Policy for real-world details.
  • Internal QA dashboards – anonymized logs containing error reports or feature flags (no personal identifiers).
  • Mock webhooks – simulated event payloads that never contain actual PII.
🔍 TESTING NOTE: Any third-party service calls are only made toward endpoints explicitly defined in Facebook's developer sandbox (e.g., graph.facebook.com/v21.0/me?mock=1). No data is sold, licensed, or transferred.

4. Your Choices & Control (Mock Consent UI)

This mock privacy policy provides a demonstration of a consent management interface similar to what users might experience on Facebook. Testing participants (developers/QA engineers) can:

  • Simulate opt-out: Use the “Reject optional mock data” button in the test harness (not shown on this static page, but part of the surrounding Facebook test suite).
  • Request mock data export: A simulated data download generates a sample JSON file with placeholder attributes for test validation.
  • Request deletion: In the test environment, deletion resets the mock user state without affecting real accounts.

Since this policy is a mock, actual users (non-testers) will never encounter this document on production Facebook surfaces.

5. Data Security (Simulated)

We follow basic security conventions for the testing environment — TLS encryption for any simulated API calls, no logging of secrets, and restricted access to internal test servers. However, as a mock environment, no real-world security guarantees are implied. Testers are responsible for using Facebook-provided test user credentials and avoiding accidental real user data.

6. International Data Transfers (Mock Clause)

For testing scenarios that involve multiple regions, this mock environment may simulate data flows consistent with standard contractual clauses. In reality, all demo data stays within the isolated test harness and never leaves the developer's local browser storage or Facebook’s test sandbox.

7. Children's Privacy (Mock Compliance)

This mock policy and associated test pages are not directed to individuals under the age of 13. No data, real or simulated, is knowingly collected from children. If you are testing COPPA-related flows, please use designated test accounts without any real child attributes.

8. Changes to This Mock Policy

As the Facebook testing requirements evolve, this mock privacy policy may be updated to reflect new UI patterns or consent checklists. The "Last Revised" date at the top of this document will indicate the latest version. Changes are effective immediately for test environments.
